# VedaAI — Assessment Creator

An AI-powered question paper generator for educators. Configure your assignment, submit it, and receive a beautifully structured exam paper — with sections, difficulty tags, and marks — generated by Groq LLM in real time.

## Architecture Overview

```
┌─────────────────────────────────────────────────────────────┐
│                        FRONTEND (Next.js)                    │
│                                                              │
│  Dashboard ──► Create Form ──► Generation Page ──► Result   │
│                    │                  ▲                      │
│               Zustand Store      WebSocket                   │
│                    │                  │                      │
└────────────────────┼──────────────────┼─────────────────────┘
                     │ REST             │ WS
                     ▼                  │
┌─────────────────────────────────────────────────────────────┐
│                     BACKEND (Express + WS)                   │
│                                                              │
│  POST /api/assignments ──► BullMQ Queue ──► Worker          │
│  GET  /api/assignments/:id               │                   │
│  POST /api/assignments/:id/regenerate    │                   │
│                                          ▼                   │
│  WebSocket Server ◄── Redis Pub/Sub ── Worker               │
│                                          │                   │
│  Redis (cache + job state)               │                   │
│  MongoDB (assignments + results)         │                   │
│                                          ▼                   │
│                                    Groq LLM API              │
│                               (llama-3.3-70b-versatile)      │
└─────────────────────────────────────────────────────────────┘
```

### Data Flow

1. Teacher fills in the Create form → submits
2. Express API validates input, saves to **MongoDB** (status: pending)
3. Job added to **BullMQ** queue (backed by Redis)
4. **Worker** picks up job:
   - Updates status → `processing`
   - Publishes progress events to **Redis Pub/Sub**
   - Calls **Groq LLM API** with structured prompt
   - Parses and validates JSON response
   - Saves `generatedPaper` to MongoDB
   - Publishes `completed` event
5. **WebSocket Server** subscribes to Redis channel → forwards events to subscribed frontend clients
6. Frontend receives real-time updates → navigates to Result page

## Getting Started

### Prerequisites
- Node.js 18+
- Docker & Docker Compose
- A free [Groq API key](https://console.groq.com)

### 1. Clone & Install

```bash
git clone https://github.com/yourusername/ai-assessment-creator.git
cd ai-assessment-creator
npm run install:all
```

### 2. Start Infrastructure

```bash
# Starts MongoDB (port 27017) and Redis (port 6379)
npm run dev:infra
```

### 3. Configure Backend

```bash
cd backend
cp .env.example .env
# Edit .env and set your GROQ_API_KEY
```

```env
PORT=4000
MONGODB_URI=mongodb://localhost:27017/ai-assessment
REDIS_URL=redis://localhost:6379
GROQ_API_KEY=gsk_your_key_here
FRONTEND_URL=http://localhost:3000
```

### 4. Configure Frontend

```bash
cd frontend
cp .env.local.example .env.local
```

### 5. Run All Services

Open **3 terminal windows**:

**Terminal 1 — API Server:**
```bash
cd backend && npm run dev
# → http://localhost:4000
```

**Terminal 2 — Worker:**
```bash
cd backend && npm run worker
# → Listens for BullMQ jobs
```

**Terminal 3 — Frontend:**
```bash
cd frontend && npm run dev
# → http://localhost:3000
```

## API Reference

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/api/assignments` | Create assignment & queue job |


## Design Decisions

### Prompt Engineering
The AI prompt is carefully structured to ensure:
- JSON-only response (no markdown, no preamble)
- Exact question count and mark total matching
- Proper section distribution based on question types
- Difficulty distribution (easy/medium/hard or mixed)
- Grade-appropriate language

### Caching Strategy
- Assignment results cached in Redis for 1 hour
- Assignment list cached for 60 seconds
- Cache invalidated on create, regenerate, delete

### Parsing & Validation
AI responses are never rendered raw:
1. JSON is extracted and cleaned
2. All fields validated and defaulted
3. Sections and questions normalized
4. Types constrained to enum values

### WebSocket + Polling
The frontend uses WebSocket as the primary real-time channel, with a polling fallback every 4 seconds to handle missed messages or reconnection gaps.
